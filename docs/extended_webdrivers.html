wait_for_stable(pause=0.0, poll_rate=0.5, timeout=10)[source]¶

Goes through a series of checks to verify the the web page is ready for use. Selenium does a majority of this but this adds extended functions by checking jQuery, Angular and the document ready state.

Parameters

  • pause (float) – The amount of time in seconds to pause code execution before checking the web page. (Default: 0.0)

  • poll_rate (float) – How often in seconds to check the state of the web page. (Default: 0.5)

  • timeout (int) – The amount of time in seconds to allow the web page to report as not ready until bypassing. If this occurs, technically there can be an issue, but code execution will continue without raising an exception. (Default: 10)

Return type

None
